# WBS (Work Breakdown Structure)

## 1. 프로젝트 개요
- 프로젝트명: PowerBuilder AS-IS 분석기 구축
- 기준 일정: 2026-02-16 ~ 2026-04-24 (10주, 2주 스프린트 5회)
- 목표: MVP 분석 파이프라인 및 핵심 리포트 제공

## 2. 작업 분해 구조

### 2.1 착수/요건 정렬
- WBS-1.1 이해관계자 인터뷰 및 범위 확정
- WBS-1.2 분석 대상 시스템 인벤토리 수집
- WBS-1.3 PRD/TRD 승인

### 2.2 추출 파이프라인 구축
- WBS-2.1 ORCA Script 추출 절차 설계
- WBS-2.2 추출 자동화 스크립트 작성
- WBS-2.3 추출 결과 manifest 규격 확정
- WBS-2.4 추출 실패/오류 처리 로직 구현

### 2.3 분석 엔진 개발
- WBS-3.1 토크나이저/파서 MVP 개발
- WBS-3.2 이벤트/함수/호출 관계 추출
- WBS-3.3 화면 전환(`opens`) 탐지
- WBS-3.4 DataWindow SQL 추출
- WBS-3.5 Embedded SQL 추출/정규화
- WBS-3.6 IR 적재 모듈 구현

### 2.4 리포팅/조회 기능
- WBS-4.1 화면 인벤토리 리포트
- WBS-4.2 이벤트-함수 맵 리포트
- WBS-4.3 테이블 영향도 리포트
- WBS-4.4 화면 이동/호출 그래프
- WBS-4.5 미사용 객체 후보 리포트

### 2.5 검증/안정화
- WBS-5.1 골든셋(20화면) 라벨링
- WBS-5.2 정확도 측정(정밀도/재현율)
- WBS-5.3 오탐/미탐 보정
- WBS-5.4 회귀 테스트 자동화
- WBS-5.5 성능 테스트 및 튜닝

### 2.6 운영 전환
- WBS-6.1 배치 실행 스크립트/CI 연동
- WBS-6.2 Runbook/운영 가이드 작성
- WBS-6.3 인수인계 및 사용자 교육

## 5. 마일스톤
- M1 (S1 종료): 추출 규격/IR 스키마 확정
- M2 (S2 종료): 호출/화면 이동 추출 MVP
- M3 (S3 종료): 핵심 리포트 3종 배포 가능
- M4 (S4 종료): 정확도 보정 완료
- M5 (S5 종료): 운영 배치 전환 완료

## 6. 품질 게이트
- QG-01: 파이프라인 E2E 3회 연속 성공
- QG-02: 정밀도 85%+, 재현율 75%+
- QG-03: 기준 데이터셋 30분 내 완료(목표)
- QG-04: Sev1/Sev2 결함 미해결 0건

## 7. 역할/책임 (RACI 요약)
| 작업영역 | Responsible | Accountable | Consulted | Informed |
|---|---|---|---|---|
| 요건/범위 | 분석 리드 | PM | 운영/현업 | 개발/QA |
| 추출/분석 엔진 | 개발 리드 | 아키텍트 | 운영 | PM/QA |
| 테스트/품질 | QA 리드 | PM | 개발 리드 | 이해관계자 |
| 운영 전환 | 운영 담당 | PM | 개발/QA | 전체 |

- 운영 전환 버퍼: 10%

- WBS-M2: CI에서 동일 버전으로 검증(로컬/CI 재현성 일치)

### 9.2 벤더 의존성 격리
- WBS-M3: ORCA 연동부 Adapter 인터페이스 분리
- WBS-M4: 추출 실패 유형 표준화(환경/권한/스크립트/라이선스)

### 9.3 규칙 거버넌스
- WBS-M5: 규칙 등록대장 운영(rule_id/version/owner/risk/regression_result)

### 9.5 운영 표준화
